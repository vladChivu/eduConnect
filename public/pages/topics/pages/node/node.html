What is Node.js? Source: https://www.javabrains.io

What is Node.js? The definition given by its creators is: "Node.js is a JavaScript runtime built on Chrome's V8 engine." We can say that the makers of Node.js took Javascript which is normally confined to a web browser and they allowed it to run on your computer (normally JavaScript runs in the browser and it can only access your web page).

How does it work? Source: https://www.javabrains.io

What's a JavaScript runtime? To understand this we need to understant the concepts of Language vs Runtime but firstly let's look at why was JavaScript created? It was created mainly to achieve some dynamic functionality in the browser because the web-browsers were only serving static files like HTML, CSS.

How is a language created?
Creating a new language in two easy steps:

  • design the language (figuring out the syntax)
  • build something that runs it (in this case enable the browser to execute the code)

JavaScript Engine?
This is where the JavaScript engine comes into the picture. This runtime can't run in isolation, it needs a context. In the case of web browser that context is the DOM, which is an object who represents the page that is rendered. So the JavaScript engine has access to the DOM and when we pass a .js file to it we have access to it and we are able to modify it's elements.

The V8 Engine
Different browsers have different implementations of JavaScript engines. We will discuss the Chrome Browser which has the V8 engine.
So V8 has access to only one context, the DOM. But this context is fixed and we might want to have access to other processes or the file system. Node.js is able to access these things (system resources, memory, file system, I/O, network and more).

In a nutshell Node.js leverages this V8 engine to run JavaScript files in a context.
