What is Express? Source: https://academind.com

As developers our main focus should be business logic and not the repetitive tasks. So we use frameworks for that because even for the simplest application it will be a lot of code for just getting the path and send a response back to the client.

When writing a web application things could get easily complicated, and error-prone because we have to write everything on our own (parsing URLs, creating routing actions, all of the checks, rendering, and more).

Express is built upon Node.js and it offers us a lot of tools which make writing node apps more easier because it implements a lot of functionalities that will take care about that nitty-gritty stuff. It is also optimized and secure.

Express allows us:

  • to quicly set up routes
  • to easily render stuff
  • to easily use templating engines (dynamically inject values into HTML files)
  • and more...

Examples

const url = require('url');
const fs = require('fs');
function renderHTML(path, response) {
   fs.readFile(path, null, function(error, data) {
      if (error) {
         response.writeHead(404);
         response.write('File not found!');
      } else {
         response.write(data);
      }
      response.end();
   });
}
module.exports = {
   handleRequest: function(request, response) {
      response:writeHead(200, {'Content-Type':
      'text/html'});
      const path = url.parse(request.url).pathname;
      switch (path) {
         case '/':
            renderHTML('./index.html', response);
            break;
         case '/login':
            renderHTML('./login.html', response)
            break;
         default:
            response.writeHead(404);
            response.write('Route not defined');
            response.end();
      }
   }
};

So instead of the above code we use express like so:

const express = require("express");
const app = express();
app.get("/welcome", (req, res) => {
   res.send(data);
});
